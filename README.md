## Featured Enterprise Project

### AI-Powered Care Home Management & Billing Automation Platform

Designed and built a **multi-tenant Agentic RAG platform** for an aged-care SaaS product.

The platform supports invoice and billing assistance, funding-schedule interpretation, backdated billing corrections, remittance reconciliation, policy question answering, document Q&A, and secure tenant-specific knowledge retrieval.

**Architecture highlights**

- FastAPI microservices for AI and document-processing workflows
- LangGraph-based RAG agent with tool calling and controlled execution
- Tenant-isolated MongoDB databases for secure multi-tenant access
- Azure OpenAI for LLM and embedding capabilities
- Azure Document Intelligence for PDF, DOCX, XLSX, and image extraction
- Microsoft Graph-based email and attachment ingestion
- Azure Service Bus with retries, backoff, and dead-letter handling
- JWT-secured APIs and server-sent events for streaming responses and progress updates
- RAGAS evaluation, Langfuse observability, MCP-based tool integration, and Dockerized Azure deployment

## How I Design AI Systems

I like to start with the workflow, not the model.

| Question | Why it matters |
| --- | --- |
| Business value | What decision, workflow, or user problem should improve? |
| Grounding | Which trusted data should the system use? |
| Control | What can the AI do directly, and what needs approval? |
| Evaluation | How will retrieval, generation, agent behavior, and outcomes be measured? |
| Reliability | How will latency, retries, failures, cost, and scale be handled? |
| Security | How will tenant isolation, authentication, authorization, and tool access be enforced? |
